## 2026-02-25 (Session 3)

**Fix**: Adaptive quality gate replaces hardcoded language overrides (INV-R06)

**Problem**: isValidMovie() had a hardcoded `smallCatalogLanguages` set (hi, ta, te, ml, kn,
mr, bn, pa, gu, ur) that unconditionally lowered the goodscore threshold by 10. This:
- Was language-discriminatory (punished users watching smaller-catalog languages)
- Applied even when the catalog had plenty of quality content
- Couldn't be tested or tuned per-deployment

**Solution**: Replaced with an adaptive quality gate that checks actual candidate availability
at runtime:
1. Compute basic pool (movies passing language + platform + availability filters)
2. Count how many pass the strict tier gate (e.g., 7.5 rating / 2000 votes for new users)
3. If >= 10 pass → use strict gate (no relaxation needed)
4. If < 10 → relax ONE step: rating -0.5, votes /2 (never below 6.5/300 absolute floor)

**Key design**: Language-neutral. A Hindi-only user with 50 quality Hindi movies gets strict
gate. Same user with only 3 quality Hindi movies gets relaxed gate. English users with thin
catalogs also get relaxed. Behavior depends on actual data, not language identity.
